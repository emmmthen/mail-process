# TDD 模板

这里存放技术方案文档模板和正式 TDD。

建议每份 TDD 对应一份 PRD，重点回答这些问题：

- 怎么实现
- 系统怎么拆
- 数据怎么流转
- 接口怎么定义
- 异常怎么处理
- 如何监控与回溯

## 推荐结构

```md
# 技术方案名称

## 1. 背景
- 对应的 PRD
- 需要解决的技术问题

## 2. 方案概述
- 总体架构
- 关键流程
- 关键模块

## 3. 输入输出定义
- 输入格式
- 输出格式
- 中间表示

## 4. 详细设计
- 模块拆分
- 数据结构
- 接口定义
- 状态流转

## 5. 核心实现
- 关键算法 / 规则
- LLM 接入方式
- Prompt / Schema / 校验

## 6. 异常与容错
- 失败处理
- 降级策略
- 重试机制

## 7. 性能与成本
- 延迟
- 吞吐
- Token / 调用成本

## 8. 监控与日志
- 指标
- Trace
- 审计日志

## 9. 风险与权衡
- 备选方案
- 为什么选这个
- 未覆盖风险
```
