# PRD 模板

这里存放产品需求文档模板和正式 PRD。

建议每份 PRD 聚焦一个产品主题，回答这些问题：

- 要解决什么问题
- 谁在使用
- 为什么现在要做
- 核心流程是什么
- 成功标准是什么
- 交互和边界在哪里
- 哪些不做

## 推荐结构

```md
# 产品名称

## 1. 背景与问题
- 现状
- 痛点
- 为什么现在要做

## 2. 目标
- 业务目标
- 用户目标
- 成功指标

## 3. 非目标
- 明确不做什么

## 4. 用户与场景
- 用户角色
- 典型场景
- 例外场景

## 5. 核心流程
- 输入
- 处理
- 输出
- 人工介入点

## 6. 功能需求
- 功能点 1
- 功能点 2
- 功能点 3

## 7. 反馈与闭环
- 如何记录反馈
- 如何回流
- 如何增强 / 削弱 / 分流

## 8. 验收标准
- 功能验收
- 体验验收
- 指标验收

## 9. 风险与待确认项
- 风险
- 依赖
- 待确认
```
